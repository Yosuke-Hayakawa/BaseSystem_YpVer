1. はじめに 
1.1. 概要 
本資料は、vTESTstudio の機能の一つであるTest Table をベースに、テスト実装を効率的にどのように行
うのか分かりやすくまとめたコマンド逆引きテクニック集です。やりたいことから実装方法を逆引きできるよう、
「○○したい」という観点からタイトルを付けており、vTESTstudio を初めて使われる方や、さまざまな機能を使
いこなしたいというエキスパートの方まで、幅広いニーズに応えられる内容となっております。 
また、「コマンド逆引き」という題名ではあるものの、掲載している内容はCAPL コードを使用したテストも含ま
れており、テスト実装全体において役立つ内容となっております。ただし、すべてのテストを網羅してはおらず
CAN のテストをベースとして作成しております。 
なお本資料は、vTESTstudio のアップデートなどにより更新されます。ご使用時には最新版をご使用くださ
い。最新版は、ベクターのお客様向け無料会員サイト「ベクター・ジャパンMember Area」に掲載いたします。 
 
1.2. 作成環境 
すべてのバージョンで共通して使えるようには作成しておりますが、一部バージョン依存のものもございます。
あらかじめご了承ください。

説明

プロパティー

使用コマンド 
名前 
Stimulate Ramp 
説明 
Stimulate Ramp テストコマンドは、CAN/LIN/FlexRay シグナル、環境変数、システム変数
を、指定された時間の間、ramp 関数によって刺激します。 
プロパティー 
プロパティー 
説明 
Caption（オプション） 
コマンドの名前。この名前はテストレポートに表示されます。 
Time Up 
ramp 波形の上限値までの立ち上がり時間 
Time High 
ramp 波形の上限値での継続時間 
Time Down 
ramp 波形の下限値までの立ち下がり時間 
Time Low 
ramp 波形の下限値での継続時間 
Duration 
ramp 波形の実行時間 
Sampling Time 
値を更新する時間 
Value A 
（オプション） 
ramp 波形の下限値。値が指定されない場合は、データベース値が使
用されます。 
Value B  
（オプション） 
ramp 波形の上限値。値が指定されない場合は、データベース値が使
用されます。 
Wait Until Finished 
設定した場合、刺激が完了するまで次のコマンドの実行が開始され
ません。設定しない場合、刺激が開始された直後に次のコマンドが
実行されます。 
 
■使用例：システム変数に、ramp の刺激として下限値0 上限値10 の間で、立ち上り時間5 秒で、上限値
で1 秒継続したあと1 秒で下限値に立ち下り、下限値の状態で1 秒間継続するのこぎり波を作成し、その波
形を20 秒継続したい。ただし、値の更新は0.5 秒とする 
① コマンド選択画面から、[Stimulate Ramp]を選択します。

4.14. ノードから送信されるすべてのTx メッセージの最小送信間隔を確認したい 
使用コマンド 
名前 
Occurrence Distance 
種類 
Background Check 
説明 
このチェックを使用すると、1 つのバス上のノードから送信されるすべてのTx メッセージの最小送
信間隔を監視できます。ノードの2 つのメッセージ間の時間間隔がMinimum Time を下回った場
合は、チェック条件をパスできません。 
このチェックの動作は、オプションプロパティーRating Period とAllowed Violations に応じて異

 Allowed Violations のみ指定した場合: 
最小間隔を下回った回数が最大許容回数を上回った場合、チェックはフェイルとなります。 
 Allowed Violations (> 0) とRating Period を指定した場合: 
タイムスロット内で、最小送信間隔を下回った回数を数えます。タイムスロット内で、最小間隔を
下回った回数が最大許容回数を超えれば、チェックはフェイルとなります。 
 Rating Period を指定し、Allowed Violations を指定しない（または = 0） 場合: 
Rating Period は無視されます。どちらの属性も指定しなかった場合と同様になります。

説明

10.1. Background Check 
Background Check は、作成時点から現在のコンテキスト（そのチェックが定義されているTest Case、テスト
シーケンス、またはテストフィクスチャー）の終了まで有効になります。コンテキストエレメントが終了する前に手
動で無効にしたい場合は、目的の位置でCheck Deactivation コマンドを実行します。（詳細は、9.1 節をご確
認ください） 
Background Check は、Test Case、テストシーケンス、またはテストフィクスチャーのPreparation セクショ
ン内で作成された場合には、このコンテキストが終了するまで有効になります。 
テストフィクスチャーにはメインシーケンスが存在しないため、テストフィクスチャーの終了まで有効になる
Background Check は、Preparation セクション内にしか定義できません。
