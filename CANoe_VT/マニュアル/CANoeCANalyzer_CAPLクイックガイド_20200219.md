文字定数と文字列定数 
CAPL では、文字は定数として記述します。1 バイ
ト文字の場合は、シングルコーテーション ‘ で囲
みます。 
  例： 
char letter = ‘A’; 
CAPLでは、ASCII文字を16進数表記で使用する
こともできます。例えば、‘A’の16進数表記は0x41
です。 
char letter = 0x41; 
配列と文字列 
配列 (Array) はすべて同じ名前のデータ項目の
集まりです。配列名以降の角かっこ内の値は、配
列サイズ、またはデータ位置を指定するインデック
ス値を示します。配列は必ずインデックス0 で始ま
ります。 
例： 
int data[3] = { 10, 20, 40 }; 
結果： 
data [0] = 10, data[1] = 20, data[2] = 40 
文字列 (Strings) 定数は、ダブルコーテーションで
囲まれる1 つ以上連続する文字で構成されます。 
例： 
char string[12] = “Hello World!”; 
文字列の最後の要素は、文字列の終端を示すNull
文字 (\0) です。そのため、配列サイズは、必ず文
字数に1 を加えた数になります。

キーワードTHIS 
キーワードthis はイベントプロシージャーで使用さ
れ、戻り値を参照するダイナミックなローカル変数
です。イベントには、メッセージ、環境変数、キー
ボード、エラーフレーム、CAN コントローラーのイ
ベントがあります。 
例： 
on message msgname  
{  
write (“signal = %d”, this.signalname); 
} 
上の例では、キーワードthis を使用して、msgname
メッセージイベントの直前のインスタンスを参照し
ます。ここで、msgname とsignalname は、データベー
スに定義される2 つの変数です。 
データにアクセスするためのイベントを参照する変
数の宣言と定義は、次のようには行えません。 
間違いの例： 
on message msgname  
  { 
message msgname mname; 
write(“signal = %d”, mname.signalname); 
  } 
上の例では、メッセージ変数mname が新しく宣言さ
れていますが、イベントから何のデータも取得しま
せん。

CAPL とは? 
CAPL (Communication Access Programming 
Language)は、システム開発やソフトウェアモ
ジュールのシミュレーション、テスト、解析をするシ
ステム開発者の要求を満たすCANoe/CANalyzer
の専用プログラミング言語です。 
 
 
 
 
 
CAPL プログラム構造の基礎 
CAPL はイベント（現象）によって制御される言語
です。それぞれのイベントにより呼び出されたプロ
グラムは、お互いを中断させることなく呼び出され
た順番に実行されます。CAPL プログラムは、イベ
ントプロシージャーの組み合わせによって構成さ
れます。以下はイベントプロシージャーの基本例で
す。 
1. (event) <event procedure name> 
2. { 
3.     write (“Hello World!”); 
4. } 
1 行目：(event)には、関数が起動するイベントタイプ
が定義されます。<event procedure name>には、参照さ
れている変数またはイベントの名前が入ります。 
2 行目と4 行目：1 行目のイベント発生時に実行さ
れるコードを{ }内に記述します。 
3 行目：1 行目で定義したイベントに反応して実行
されるコードをここに記述します。 
コメント 
プログラム内のコードセクションにコメント行を入れ
ることができます。コメントが、1 行の場合 // をコメ
ントの先頭に記述し、複数行にまたがる場合は /* 
と */ を入力して、その間にコメントを記述します。 
大文字と小文字の区別 
CAPL プログラムで宣言される変数とデータベース
に参照される変数は、すべて大文字と小文字が区
別されます。ユーザー定義関数とCAPL-DLL で定
義される関数も、すべて大文字と小文字が区別さ
れます。すべての組込関数とキーワードに関して
は、大文字と小文字が区別されません。

IF 文 
if 文を使用して式を評価し、評価結果が「真 (True)」
の場合のみ命令が実行されます。評価結果が「偽
(False)」の場合、コードブロックは実行されません。 
一般的な形式： 
if (条件文) 
{ ...条件文の結果がTrue の場合に実行される命令文... } 
例： 
if (Blinker_On == 1) 
{ 
write (“Blinker is on.”); 
} 
注記：if 文はネストする（=入れ子構造にする）こと
ができます。例えばif 文の中にさらに（1つ以上の）
if 文を記述することができます。 
IF-ELSE 文 
if-else 文も式を評価できます。評価した式の結果に
よって、2 つの命令のいずれかが実行されるという
点がif 文と異なります。式の結果が「真 (True)」であ
ればif 節の命令が実行され、結果が「偽 (False)」で
あれば、else 節の命令が実行されます。 
ステートメント（命令文） 
一般的な形式： 
if ( 条件文 ) 
{ ...条件文の結果がTrue の時に実行される命令文... } 
else 
{ ...条件文の結果がFalse の時に実行される命令文... } 
 
例： 
// 時計の「分」を進める関数 
increment_clock_minutes() 
{ 
int clock_minutes;

     clock_minutes = 0; 
else

} 
注記：if-else文も、if文同様にネストする（= 入れ子
構造にする）ことができ、たとえばif節の中にさらに
if-else 文を、else 節の中にさらにif-else 文を記述す
ることができます。

switch 文は、if-else 文と同じ制御になりますが、2 つ
以上の選択肢を必要とする場合に使用するとより
便利です。switch 文の中にcase 文を複数記述する
ことによって1 つの文に2 つ以上の分岐をつくるこ
とができます。 
switch 文は、式の「値」に応じて実行内容を分岐し
ます。「値」と「整数または文字の定数」を比較し、
一致した選択肢の文を実行します。 
つまり、1 つ以上のcase 文と比較し、｛ ｝ 内で最
初に一致した文の制御が実行されます。case 文は
いくつあってもよく、1 つの ｛ ｝ の中にすべて記述
できます。 
case の後ろは、整数の定数、文字の定数、定数式
のいずれかになり、定数式には、変数や関数を置
くことはできません。 
例： 
float value1, value2, result; 
char operator; 
... 
switch (operator) 
{ 
case ‘+’:   result = value1 + value2; 
 
        break; 
case ‘-’:    result = value1 - value2; 
  
        break; 
case ‘*’:   result = value1 * value2; 
 
       break; 
case ‘/’:   result = value1 / value2; 
 
       break; 
} 
注記：どのcase 文にも値が一致しない場合、
default 文が実行されます。default 文がない場合
（default は省略も可能）、switch 文を抜けます。

do-while 文は、while 文と同じ制御ですが、実行中

このループ文は、最初に命令文を実行し、次に条
件式を評価します。条件式が「真 (True)」の場合、
それが「偽 (False)」になるまで命令文の始めから
繰り返し実行（ループ）されます。 
一般的な形式： 
do 
{  
...条件式の結果がTrue の場合に実行される命令文... 
} 
while (条件文); 
例：

{ 
     count ++; 
}

FOR ループ 
for 文もまた、while、do-while と同様に繰り返し実
行されるループ文の1 つです。定められた回数だ
け繰り返し実行する場合に用いると便利です。 
一般的な形式： 
for (初期設定式; 条件式; 再設定式) 
{ 
...条件式の結果がTrue の場合に実行される命令文... 
} 
注記：「初期設定式」とは、最初にfor 文が処理され
る時に1 度だけ実行される式です。「条件式」の意
味はwhile 文のものと同じで「真 (true)」の時にルー
プします。「再設定式」は、“繰り返す処理”が最後ま
で終了した時に実行される式でループする度（「繰
り返す処理」が終了する度）に実行されます。 
例： 
// 10 回ループする 
int i, count; 
for ( i = 0; i < 10; i ++ ) 
{ 
 
count ++; 
} 
注記：条件式を省略した場合、for 文は無限ループ
になりますのでご注意下さい。またCAPL ではWait
のためのfor 文は使用できません。 
BREAK 文 
break 文はswitch、do-while、while、for の繰り返し
文の中でループから抜け出すために用います。 
CONTINUE 文 
continue 文は、switch、do-while、while、for の繰り
返し文の中で、ループ中の残りの処理をスキップし
て、ループの先頭に戻ります。 
例： 
int x; 
while ( x != 0 ) 
{ 
 
doThis (x); 
 
if( x == 4 ) continue; 
 
doThat (x); 
}

を抜けることもできます。 
例： 
long Power( int x ) 
{

データベースを利用したメッセージの宣言

プログラムで使用することができます。

message 0x100x eTemp = {DLC=8};
