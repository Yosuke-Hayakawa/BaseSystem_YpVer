2 vTESTstudioについて 
vTESTstudio(TSO)は、テストシーケンスを実装するための統合環境です。 
従来のTest Automation Editor(TAE)に類似したグラフィカルユーザーインターフェイスや、 
CAPL やC#といった、さまざまなテスト設計言語を使用してテストのプログラムの記述が可能です。 
 
【各種エディター】 
・ CAPL エディター：CANoe のイベント指向のプログラミング言語 
・ C#エディター：特別に設計されたテスト用ライブラリーでCANoe を拡張する.NET プログラミング言語 
・ テストテーブルエディター：テーブル形式でプログラミングが可能なため専用知識は必要なし 
・ テストダイアグラムエディター：グラフィカルな表記法によるテストシーケンスの実装が可能 （※） 
・ ステートダイアグラムエディター：予想されるSUT の挙動を状態遷移図によってモデル化し、 
テストケースを自動生成 （※）               ※オプション製品 (Option Graphical Design) 
 
【その他の特徴および機能】 
・ 再利用コンセプト：ライブラリーや個別ファイルにより、各種テストへの再利用が可能 
・ テスト作成の構造化：テスト作成の構造化をするために、テストに複数のファイルを含むことが可能 
・ パラメーター機能：テストで利用するパラメーターを定義でき、テストでファイルの再利用が可能 
・ Stimulation curve：Waveform Editor を用いて、Stimulation curve を定義可能 
・ バリアント：テストロジック、テスト実装、パラメーター値へのバリアント機能をサポート

2.2 RQM/TDM システムとの連携 
RQM/TDM システムを利用することで、テストの実装とテストレポートの中に、外部的に定義された要件と
テスト記述を含むことができるため、テストデータの追跡が可能となっています。 
RQM/TDM システムからエクスポートした情報は、vTESTstudio にインポートし、テストケースに 
リンクさせることができます。リンクさせる要素には、要件またはテスト記述を使用します。 
これらの要素はトレーサビリティーを実現するために使用され、トレース項目として指定されます。 
RQM/TDM システムとしては、現状、Vector PREEvision、PTC Integrity、IBM DOORS、 
IBM DOORS NG/RQM、Siemens Polarion ALM などに対応したファイルをサポートしています。

ます。 
 
 
 
 
 
 
 
 
 
 
 
 
 
プロジェクトツリーでは、使用中のファイルが確認できます。 
 
例では、TestUnit1, TestUnit2, TestUnit3 はすべて同一の 
Test1.vtt が割り当てられています。

テスト内のパラメーターのみが異なるテストが実行されます。

2.4.5 
State Diagram Editor 
SUT の挙動を状態遷移図としてモデル化し、テストケースを自動生成が可能です。 
※オプション機能(Graphical Test Design) 
・ 状態遷移図モデルからテストケース自動生成 
・ 生成アルゴリズムは「チャイニーズポストマン(CPP)探索」、「幅優先探索」から選択可能 
（デフォルト: 幅優先探索） 
・ CAPL やC#で記述したテストケース／テスト関数の呼び出しが可能 
 
 
 
2.5 パラメーター 
テストシーケンスと分離させたパラメーターの定義が可能です。そのため、別ファイルで保存されます。 
異なるテストで使用できるためパラメーターの再利用が可能となっています。 
「スカラー値」、「スカラー配列」、「構造体パラメーター」、「構造体配列」を作成することができます。 
・ MS Excel から直接編集可能な標準ファイルフォーマット(CSV)に対応 
・ コーディングの際、直接パラメーターを指定することでアクセスが可能

【function】 
function は一般的な関数扱いとなるため、テストレポートは個別の結果を表示するのみです。
