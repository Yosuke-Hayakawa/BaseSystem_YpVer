この資料について 
Vector の新しいライセンスモデルとして、2018 年頃からNew Vector License Model(NVLM)が 
導入されており、この新しいライセンスモデルはVector License Client というツールで管理されます。 
このVector License Client では以下の様な作業が可能であり、その手順を本資料で案内しています。 
 
[Activate] 
ライセンスをライセンスキャリア(インターフェース / PC / dongle)に付与することが出来ます 
 
[Deactivate] 
ライセンスを一時返却することが出来ます(ライセンスキャリアを変更する際に使用します) 
 
[Reactivate] 
ライセンスの期限を更新することが出来ます(NVLM のライセンス期限は12 ヶ月です) 
 
[Migration] 
旧ライセンス形式から新ライセンス形式への移行が出来ます 
 
[License Update] 
対象ライセンスの使用可能な最大バージョンを更新することが出来ます

Vector License Client の起動方法 
Vector License Client は基本的にソフトウェア(CANoe やCANape など)をインストールすることで 
同時にインストールされていますので、Windows のスタートメニューを開き、プログラムの一覧から 
Vector License Client を選択して起動して下さい。 
 
ただ、ソフトウェアと同時にインストールされたバージョンですと古い可能性がありますので、 
以下より最新版を取得し、ご利用下さい。本資料ではV7.3 にて解説いたします。 
https://download.vector.com/addons/VectorLicenseClientInstaller.zip 
Vector License Client のコンポーネント(CodeMeter)にセキュリティの脆弱性が確認されました。 
この脆弱性はVector License Client V5.3 にて解消されますので、まだV5.3 以降のバージョンに 
更新されていない方は、必ず上記リンクから更新してご利用下さい。 
 
本件についての詳細は以下のリンク先をご参照下さい。 
https://www.vector.com/jp/ja/news/news/vjnews20100201/

作業前の確認事項 
Activate などの作業を行うPC がVector のライセンスサーバーに接続できるかを確認します。 
この結果により、作業の手順や内容が変わるので、必ず事前に行って下さい。 
 
1. 
Vector License Client を起動し、リボンメニュー内の[Troubleshooting]を押下します。 
 
2. 
下図赤枠部内の項目に赤文字で書かれたエラーが無ければオンライン作業が可能です。 
『4  オンライン作業』 の中から必要な作業手順をご確認下さい。 
 
 
 
 
赤文字で何らかのエラーが表示された場合は以下どちらかの方法をご検討下さい 
方法1 (推奨):  巻末FAQ の6.1 や6.2 を参考に、エラーを解消後、オンライン作業を行う 
方法2 (非推奨):  オフライン手順(5 章参照)にて作業を行う

4.1 
Activate (オンライン) 
この作業は、ライセンスをデバイスに印加する際に行います。 
 
【 操作手順 】 
1. 
ライセンスを付与するデバイスをPC に接続後、Vector License Client を起動し、リボンメニュー
の[Activate]ボタンを押下します (PC にライセンスを付与する場合、デバイスの接続は不要です)。 
 
2. 
以下の様な画面が開きますので、下図①の箇所にメールに添付された納品書に書かれている 
Activation Key を入力後、[Load]ボタン(②)を押下し、ライセンスが表示されることを確認します。 
表示されたライセンスからActivate したいライセンスにチェック(③)を入れ(複数の場合は、全てに 
入れます)、[License Carrier](④)でライセンス印加先を選択し、[Activate](⑤)ボタンを押下しま
す。 
 
3. 下図の様にActivate した製品名(License Name)やターゲットとしたデバイス(License Carrier)が 
正しく表示されたことを確認し、Activate は終了です。

4.2 
Deactivate (オンライン) 
この作業は、デバイスに印加されたライセンスを一時返却する際に行います。 
ライセンスを返却する目的がLicense carrier 変更の場合、Deactivate 終了後、任意のLicense carrier へActivate
することでLicense carrier の変更が完了いたします。 
Activate については、4.1 節(オンライン)もしくは5.1 節(オフライン)の手順を参照ください。 
弊社システム変更により、弊社でのライセンス開放は不要となりましたので、現在は弊社へのライセンス開放依
頼は不要となっております。 
ただし、12 ヶ月に4 回まではデバイスを変更することができますが、5 回以上は変更ができません。 
デバイスの変更は容易になりますが、この回数制限にご留意頂き運用をお願いいたします。 
回数制限についての詳細は、6.5 節をご確認ください。 
【 操作手順 】 
1. 返却したいライセンス名を選択し、[Deactivate]を押下します。 
   
 
2. [Yes]を押下します。 
 
3. 返却するライセンス名やLicense ID が正しいことを確認し、[Send Request]を押下します。

4.3 
Reactivate (オンライン) 
ライセンスの期限は基本的に12 ヶ月となっております。期限切れの30 日前からReactivate が 
可能となるので、こちらの作業を行って期限を更新して下さい。 
なお、期限が切れた後に行っても問題はございません。 
 
【 操作手順 】 
1.  ライセンスの期限が近い(もしくは既に切れている)製品名を選択し、[Reactivate]を押下します。 
 
2.  [Send Request]を押下します。 
 
3.  ライセンス期限が伸びたことを確認して終了となります。

4.4 
Migrate (オンライン) 
ライセンスビットと呼ばれる旧ライセンスがインターフェースやドングルに印加されている状態から 
新ライセンスに移行する作業のことをマイグレーションと呼んでおります。こちらの作業を行うことで、 
旧ライセンスビットを削除し、新しいライセンスを印加出来る状態にします。 
 
【 操作手順 】 
1. 
PC にマイグレーション前の旧ライセンスビットを持ったインターフェースを接続しておきます。 
2. 
リボンメニューの[Migrate]ボタンを押下します。 
 
3. 
下図赤枠部の[License Bits…]ボタンを押します。 
 
4. 
PC に接続されているインターフェースから、マイグレーション前の旧ライセンスの情報が 
表示されますので、間違いがないことを確認し、[OK]で閉じます。

4.5 
License Update (オンライン) 
製品の保守をご契約いただいている場合、新しいバージョンがリリースされると保守の特典として 
そのバージョンの使用権が与えられます。その内容のお知らせがメールで届きましたら、こちらの 
作業を行うことで、使用可能なバージョン(Maximum Version)を引き上げることが出来ます。 
 
【 操作手順 】 
1. 更新するライセンスの現在のMaximum Version を確認しておきます 
 
2. [License Update]ボタンを押下します 
 
3. Maximum Version が上がったことを確認し、終了です

オフライン作業 
この章は3 章で確認したTroubleshooting の結果、オンライン作業が不可能な場合の手順となります。 
手順が多いことから、6 章のFAQ を参考にオンライン作業を可能にしていただくことを推奨します。

そのPC だけで上記の操作を行うことが可能です(オフラインPC をVector License Client、 
オンラインPC をブラウザの操作に置き換えてご確認下さい)。

Toubleshooting 画面で”The Vector License Server can not be reached”と表示され、オンラ
