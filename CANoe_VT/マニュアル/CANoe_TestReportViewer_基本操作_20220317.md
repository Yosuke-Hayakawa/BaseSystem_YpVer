1 はじめに 
1.1 
本書の内容 
本書はCANoe に標準で付属する「Vector CANoe Test Report Viewer」（以降Test Report Viewer と呼
称）の操作方法を記述します。 
 
Test Report Viewer はCANoe テスト機能により自動生成されるテストレポートの解析を行うためのビュー
アーです。Test Report Viewer を利用することで、直感的な操作でテストレポートの結果を確認できるだけで
なく、テスト結果のフィルタリングやログファイルとの連携、テスト実装へのナビゲーションなどが可能です。 
 
本書ではまず2 章にて基本的な操作方法をチュートリアル形式で記述します。基本的な操作方法だけでなく、ク
エリの作成方法やフィルタリング機能、PDF・XML などのファイルエクスポート方法についても記述しています。
この2 章のチュートリアルを読むことで、Test Report Viewer の基本的な操作方法をひととおりご理解いただ
くことができますので、2 章は通してお読みいただくことをお勧めします。3 章ではそれ以外の有用な機能につい
て節ごとに独立した形で紹介し、最後に4 章でよくあるお問い合わせ内容を記載しています。 
 
1.2 対象読者 
本書はCANoe およびvTESTstudio の基本的な操作方法をご理解いただいている方を対象としています。こ
れらツールの基本的な操作方法につきましては、1.4 節の参考資料を参照してください。 
 
1.3 動作環境 
本書は以下のツール、およびバージョンを使用して記述します。 
 
CANoe 12.0 SP4 
 
Test Report Viewer 12.0 SP3 
 
vTESTstudio 4.0 SP3 
 
なお、本書の3 章では上記のバージョン以降で追加された機能も記載します。その場合は都度使用したバー
ジョンについて記載します。

■その他の有用なフィルタリング機能 
 
Wait コマンドのフィルタリング 
テストレポートにはテスト実行時のすべての情報が記載されますが、Wait コマンドなどで待機した情
報は一般的にはあまり確認の必要がありません。そのためTest Report Viewer ではWait コマンド
の表示をフィルタリングする機能があります。 
 
 
「Simple」は、Timeout 時間だけが設定される単純なWait コマンドをフィルタリングします。 
「Complex」は上記以外の（Timeout 時間以外に待機を終了する条件のついた）Wait コマンドをフィ
ルタリングします。これらは同時に有効にすることもできます。 
 
 
同じViolation の表示の抑制 
送信周期の確認など、vTESTstudio 上で「Background Checks」に分類されるような監視系のコマ
ンドを利用した場合、チェック条件違反が発生すると「Violation」が記録されます。監視期間中に
「Violation」が発生する場合、一般的には1 回の発生では終わらず、多数の違反が記録されることに
なります。Test Report Viewer ではこのような同じ内容の「Violation」がレポートを埋め尽くさないよ
