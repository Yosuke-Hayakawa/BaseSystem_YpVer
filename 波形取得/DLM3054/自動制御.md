# DLM3054HD & PBL100（701988）自動制御リファレンス

> 対象機器: YOKOGAWA DLM3054HD（4ch混合信号オシロスコープ）  
> ロジックプローブ: 701988（PBL100 / 8bit）  
> 通信コマンド準拠: IM DLM3054HD-17JA  
> ロジックプローブ準拠: IM 701988-01  

---

## 1. 概要

### DLM3054HD リモート制御

DLM3054HDはUSB TMC（Test & Measurement Class）経由でPCから完全に制御できる。  
横河のTMCTLライブラリ（tmctlLib.py）を使い、SCPIライクなコマンドを送受信する。

主な操作：
- チャンネル設定（電圧レンジ、カップリング、プローブ比等）
- トリガ設定（エッジ、レベル、モード）
- 波形データの取得（ASCII/バイナリ）
- 画面キャプチャ（PNG/BMP）
- ファイル操作（内蔵メモリ/USB）
- 自動計測（周波数、振幅、RMS等）
- ロジックプローブ制御（閾値、表示、バス設定）

### PBL100（ロジックプローブ 701988）

DLM3054HDのPODA/PODBポートに接続する8bit入力のロジックプローブ。  
デジタル信号の波形取得・バス解析に使用する。

| 項目 | 仕様 |
|------|------|
| 入力チャンネル数 | 8bit（Bit0～Bit7） |
| 入力閾値範囲 | -40V ～ +40V |
| 閾値プリセット | CMOS 1.5V/2.5V/3.3V/5V、ECL -1.3V |
| トグル周波数 | DC ～ 100MHz |
| 最小パルス幅 | 5ns |
| 入力インピーダンス | 1MΩ / 10pF（代表値） |
| 入力電圧範囲 | -42V ～ +42V（DC+ACピーク） |
| ケーブル長 | 約1.8m |

### 通信方式

| 項目 | 値 |
|------|-----|
| インターフェース | USB 2.0（TMCクラス） |
| 接続タイプ | `TM_CTL_USBTMC2` |
| アドレス | `393033314437363232` |
| ライブラリ | tmctlLib.py（横河公式Pythonラッパー） |

## 2. 接続・初期化

### 本体側の設定

1. DLM3054HDの `MISC` → `Remote Control` → `USB` を **TMC** に変更
2. 「Mass Storage」のままだとUSBメモリとして認識されるだけで通信できない
3. モード変更後はUSBケーブルの抜き差しが必要な場合がある

### PCドライバのインストール

1. `ソフト/YTUSB2300.zip` を展開
2. `Setup.exe` を実行してUSBドライバをインストール
3. DLM3054HDをUSB接続し、デバイスマネージャーでYOKOGAWAデバイスとして認識されることを確認

### ライブラリ配置

```
波形取得/DLM3054/lib/
├── tmctlLib.py    # Pythonラッパー
├── tmctl64.dll    # 64bit用
├── tmctl.dll      # 32bit用
├── YKMUSB64.dll   # USB通信DLL(64bit)
└── YKMUSB.dll     # USB通信DLL(32bit)
```

### 接続コード

```python
import sys, os
sys.path.insert(0, r'波形取得\DLM3054\lib')
from tmctlLib import TMCTL, TM_CTL_USBTMC2

# 接続
tm = TMCTL()
ret, dev_id = tm.Initialize(TM_CTL_USBTMC2, '393033314437363232')

# 接続確認
tm.Send(dev_id, '*IDN?')
ret, resp, rlen = tm.Receive(dev_id, 1024)
print(resp)  # YOKOGAWA,DLM3054HD,9031D7622,F1.21

# 切断
tm.Finish(dev_id)
```

### デバイス検索

```python
device_list = []
ret, num = tm.SearchDevices(TM_CTL_USBTMC2, device_list, 10, '')
for dev in device_list:
    print(f'アドレス: {dev.adr}')
```

> **注意**: `TM_CTL_USB` や `TM_CTL_USBTMC` では検出されない。必ず `TM_CTL_USBTMC2` を使うこと。

### エラーハンドリング

tmctlLibは失敗時に例外を投げる。主なエラーコード：

| コード | 内容 |
|--------|------|
| 1 | タイムアウト |
| 2 | デバイスが見つからない |
| 4 | 接続失敗 |
| 8 | 未接続状態 |
| 64 | パラメータ不正 |
| 256 | 送信エラー |
| 512 | 受信エラー |
| 1024 | ブロックデータではない |

```python
try:
    ret, dev_id = tm.Initialize(TM_CTL_USBTMC2, '393033314437363232')
except Exception as e:
    print(f'接続失敗: {e}')
```

### ヘルパー関数

以降のコード例で共通して使うヘルパー：

```python
def send(cmd):
    """コマンド送信"""
    tm.Send(dev_id, cmd)

def query(cmd, bufsize=4096):
    """コマンド送信＋応答受信"""
    tm.Send(dev_id, cmd)
    ret, resp, rlen = tm.Receive(dev_id, bufsize)
    return resp.strip()

def wait_complete():
    """オーバーラップコマンドの完了待ち"""
    resp = query('*OPC?')
    return resp == '1'
```

## 3. DLM3054 制御コマンドリファレンス

### コマンド構文の規則

- 大文字部分が最短省略形（例: `:CHANnel` → `:CHAN` でもOK）
- `[]` で囲まれた部分は省略可能
- `<x>` はチャンネル番号等のパラメータ
- `?` 付きはクエリ（現在値の問い合わせ）
- ヘッダ付き応答を無効にする場合: `:COMMunicate:HEADer OFF`

---

### 3.1 基本制御

| コマンド | 説明 | パラメータ/応答 |
|---------|------|---------------|
| `*RST` | 初期設定に戻す | — |
| `*CLS` | ステータスレジスタをクリア | — |
| `*IDN?` | 機器情報を返す | `YOKOGAWA,DLM3054HD,<シリアル>,<FW>` |
| `*OPC?` | 実行中のコマンドが完了したか | `1`=完了 |
| `*OPC` | 完了時にOPCビットをセット | — |
| `*WAI` | 実行完了まで待つ | — |
| `:STATUS:ERROR?` | 最新のエラーを返す | `<コード>,"<メッセージ>"` |
| `:COMMunicate:REMote ON\|OFF` | リモートモード切替 | `ON`=リモート |
| `:COMMunicate:HEADer ON\|OFF` | 応答にヘッダを付加するか | `OFF`推奨 |
| `:COMMunicate:LOCKout ON\|OFF` | パネルロック | `ON`=操作禁止 |
| `:COMMunicate:WAIT <NRf>` | OPC待ちの比較値 | 通常設定不要 |

---

### 3.2 実行制御

| コマンド | 説明 | パラメータ |
|---------|------|-----------|
| `:STARt` | 波形取り込み開始 | — |
| `:STOP` | 波形取り込み停止 | — |
| `:SSTart?` | シングルトリガ実行＋タイムアウト付き完了待ち | パラメータ: `<NRf>`（100ms単位）、`0`=完了、`1`=タイムアウト |
| `:ASETup:EXECute` | オートセットアップ実行 | — |
| `:INITialize` | 初期設定にリセット | — |

#### シングルトリガの実行

```python
send(':TRIGger:MODE NSINgle')
result = query(':SSTart? 100')  # 10秒タイムアウト
# 0=トリガ完了, 1=タイムアウト
```

### 3.6 波形取得

#### 波形取得コマンド

| コマンド | 説明 | パラメータ |
|---------|------|-----------|
| `:WAVeform:TRACe` | 取得対象トレース | `CHANnel<x>\|MATH<x>\|LOGIC` 等 |
| `:WAVeform:FORMat` | データ形式 | `ASCii\|BYTE\|WORD\|RBYTe` |
| `:WAVeform:STARt` | 取得開始位置 | 0 ～ レコード長-1 |
| `:WAVeform:END` | 取得終了位置 | 0 ～ レコード長-1 |
| `:WAVeform:LENGth?` | レコード長の問い合わせ | 応答: ポイント数 |
| `:WAVeform:SEND?` | 波形データ転送 | 応答: 波形データ |
| `:WAVeform:BYTeorder` | バイトオーダー（WORD時） | `LSBFirst\|MSBFirst` |
| `:WAVeform:OFFSet?` | 波形のオフセット値 | 応答: 電圧値 |
| `:WAVeform:RANGe?` | 波形のレンジ値 | 応答: 電圧値 |
| `:WAVeform:BITS?` | データのビット数 | 応答: `8\|12\|16` |
| `:WAVeform:SIGN?` | 符号付きか | 応答: `0`=符号なし、`1`=符号付き |
| `:WAVeform:SRATe?` | 波形のサンプルレート | 応答: Hz |

#### データフォーマット

| フォーマット | データ形式 | 速度 | 用途 |
|-------------|-----------|------|------|
| `ASCii` | テキスト（カンマ区切り数値） | 遅い | デバッグ・小規模取得 |
| `BYTE` | 8bitバイナリ | 速い | 高速取得（精度低） |
| `WORD` | 16bitバイナリ | 速い | 高速取得（精度高）推奨 |
| `RBYTe` | 8bitバイナリ（参照） | 速い | 未加工データ |

#### 物理値変換式

バイナリデータから物理値（電圧等）への変換：

**BYTE フォーマット（8bit）:**
```
物理値 = (rawデータ - 128) / 128 × Range / 2 + Offset
```

**WORD フォーマット（16bit）:**
```
物理値 = rawデータ / 32768 × Range / 2 + Offset
```

ここで：
- `Range` = `:WAVeform:RANGe?` の応答値
- `Offset` = `:WAVeform:OFFSet?` の応答値
- `rawデータ` = バイナリから変換された整数値（符号付き）

#### ブロックデータ形式

バイナリ転送時の応答は IEEE 488.2 ブロックデータ形式：

```
#<桁数><バイト数><データ本体>
```

例: `#41000<1000バイトのデータ>` → 桁数=4、バイト数=1000

tmctlLibの `ReceiveBlockHeader()` + `ReceiveBlockData()` で処理できる。

### 3.7 画面キャプチャ

#### PCへの直接転送

```python
send(':IMAGe:FORMat PNG')
send(':IMAGe:TONE COLor')
send(':IMAGe:INFormation OFF')   # 設定情報の付加
send(':IMAGe:SEND?')

# ブロックヘッダ受信
ret, length = tm.ReceiveBlockHeader(dev_id)

# 画像データ受信
buf = bytearray(length)
ret, rlen, end = tm.ReceiveBlockData(dev_id, buf, length)

with open('screenshot.png', 'wb') as f:
    f.write(buf[:rlen])
```

#### 内蔵メモリ/USBへの保存

```python
send(':IMAGe:FORMat PNG')
send(':IMAGe:TONE COLor')

# 保存先ドライブ設定（USBメモリ）
send(':IMAGe:SAVE:DRIVe USB,0')

# ファイル名自動付番
send(':IMAGe:SAVE:ANAMing DATE')

# 保存実行
send(':IMAGe:EXECute')
wait_complete()
```

#### 画像フォーマット

| パラメータ | 説明 | 値 |
|-----------|------|-----|
| `:IMAGe:FORMat` | ファイル形式 | `PNG\|BMP\|JPEG` |
| `:IMAGe:TONE` | 色調 | `COLor\|GRAY\|OFF\|REVerse` |
| `:IMAGe:INFormation` | 設定情報の付加 | `ON\|OFF` |

### 3.9 測定機能

#### 自動計測の設定と実行

```python
# 測定パラメータON

send(':MEASure:CHANnel1:AMPLitude:STATe ON')

# 値取得

amp = query(':MEASure:CHANnel1:AMPLitude:VALue?')

# 統計値
mean = query(':MEASure:CHANnel1:FREQuency:MEAN?')
sdev = query(':MEASure:CHANnel1:FREQuency:SDEViation?')
```

## 5. 実践的な使用例（Pythonコード）

### 5.1 基本セットアップ

接続→リセット→チャンネル設定→トリガ設定→時間軸設定の一連の流れ。

```python
import sys, os
sys.path.insert(0, r'波形取得\DLM3054\lib')
from tmctlLib import TMCTL, TM_CTL_USBTMC2

ADDRESS = '393033314437363232'

# --- 接続 ---
tm = TMCTL()
ret, dev_id = tm.Initialize(TM_CTL_USBTMC2, ADDRESS)

def send(cmd):
    tm.Send(dev_id, cmd)

def query(cmd, bufsize=4096):
    tm.Send(dev_id, cmd)
    ret, resp, rlen = tm.Receive(dev_id, bufsize)
    return resp.strip()

# --- 初期化 ---
send('*RST')
send('*CLS')
send(':COMMunicate:HEADer OFF')     # ヘッダなし応答
send(':COMMunicate:LOCKout ON')      # パネルロック

# 接続確認
idn = query('*IDN?')
print(f'接続先: {idn}')

# --- CH1設定（+B電源監視）---
send(':CHANnel1:DISPlay ON')
send(':CHANnel1:COUPling DC')
send(':CHANnel1:VDIV 2')             # 2V/div
send(':CHANnel1:OFFSet 6')           # オフセット6V（中心を6Vに）
send(':CHANnel1:PROBe 1')            # 1:1プローブ
send(':CHANnel1:LABel "+B"')
send(':CHANnel1:LABel:STATe ON')

# --- CH2設定（CLG1監視）---
send(':CHANnel2:DISPlay ON')
send(':CHANnel2:COUPling DC')
send(':CHANnel2:VDIV 2')
send(':CHANnel2:OFFSet 6')
send(':CHANnel2:PROBe 1')
send(':CHANnel2:LABel "CLG1"')
send(':CHANnel2:LABel:STATe ON')

# --- CH3, CH4 OFF ---
send(':CHANnel3:DISPlay OFF')
send(':CHANnel4:DISPlay OFF')

# --- トリガ設定（CH1立ち上がりエッジ）---
send(':TRIGger:MODE NORMal')
send(':TRIGger:SIMPle:SOURce CHANnel1')
```

### 5.2 波形取得と保存（ASCII）

シングルトリガで波形を取得し、CSVとして保存する。

```python
import time, csv

send(':TRIGger:MODE NSINgle')

send(f':WAVeform:END {length - 1}')

with open(output_file, 'w', newline='') as f:
    writer = csv.writer(f)

    for i, v in enumerate(values):
```

### 5.3 波形取得（バイナリ高速転送）

WORDフォーマットで高速に波形データを取得し、物理値に変換する。

```python
import struct

# 取り込み停止状態で実行すること
send(':WAVeform:TRACe CHANnel1')
send(':WAVeform:FORMat WORD')
send(':WAVeform:BYTeorder LSBFirst')
send(':WAVeform:STARt 0')

length = int(query(':WAVeform:LENGth?'))
send(f':WAVeform:END {length - 1}')

# 変換に必要なパラメータ取得
vrange = float(query(':WAVeform:RANGe?'))
voffset = float(query(':WAVeform:OFFSet?'))
srate = float(query(':WAVeform:SRATe?'))

# バイナリデータ転送要求
send(':WAVeform:SEND?')

# ブロックヘッダ受信
ret, data_length = tm.ReceiveBlockHeader(dev_id)

# データ受信
buf = bytearray(data_length)
total_received = 0
while total_received < data_length:
    remaining = data_length - total_received
    chunk_size = min(remaining, 1048576)  # 1MBずつ
    temp = bytearray(chunk_size)
    ret, rlen, end = tm.ReceiveBlockData(dev_id, temp, chunk_size)
    buf[total_received:total_received + rlen] = temp[:rlen]
    total_received += rlen
    if end:
        break

# 16bit符号付き整数に変換
num_points = data_length // 2
raw_data = struct.unpack(f'<{num_points}h', buf[:num_points * 2])

# 物理値変換: V = raw / 32768 × Range / 2 + Offset
voltages = [(r / 32768.0) * (vrange / 2.0) + voffset for r in raw_data]

dt = 1.0 / srate
print(f'取得完了: {num_points}点, Fs={srate/1e6:.1f}MHz, Range={vrange}V, Offset={voffset}V')

# CSV保存
import csv
with open('waveform_binary.csv', 'w', newline='') as f:
    writer = csv.writer(f)
    writer.writerow(['Time(s)', 'CH1(V)'])
    for i, v in enumerate(voltages):
        writer.writerow([f'{i * dt:.9e}', f'{v:.6e}'])
```

## 6. 注意事項・トラブルシューティング

### USB接続

| 症状 | 原因 | 対処 |
|------|------|------|
| デバイスが見つからない | USBモードがMass Storage | 本体: MISC → Remote Control → USB → **TMC** に変更 |
| Initialize失敗 | 接続タイプが違う | `TM_CTL_USBTMC2` を使う（USBやUSBTMCでは不可） |
| 通信タイムアウト | タイムアウト短すぎ | `tm.SetTimeout(dev_id, 30000)` で延長（ms単位） |
| ドライバ未インストール | YTUSB2300未導入 | `ソフト/YTUSB2300.zip` からインストール |

### ヘッダ付き応答

デフォルトでは応答にコマンドヘッダが付加される場合がある。

```
:CHANNEL1:VDIV 2.000000E+00    ← ヘッダ付き
2.000000E+00                    ← ヘッダなし
```

値のパースが面倒になるため、接続直後に必ず以下を送ること：

```python
send(':COMMunicate:HEADer OFF')
```

### オーバーラップコマンドの完了待ち

一部コマンド（`:ASETup:EXECute`、`:FILE:SAVE:*:EXECute`等）はオーバーラップ実行される。  
次のコマンドを送る前に完了を確認すること。

```python
send(':FILE:SAVE:BINary:EXECute')
# 方法1: *OPC? で完了待ち
resp = query('*OPC?')  # '1' が返るまでブロック

# 方法2: *WAI で待機
send(':FILE:SAVE:BINary:EXECute;*WAI')
```

### サンプルレートの自動決定

DLM3054HDのサンプルレートはTDIV（時間軸）とレコード長から自動計算される。

```
サンプルレート ≒ レコード長 / (TDIV × 10)
```

明示指定も可能だが、通常はAUTOのままで問題ない。  
現在のサンプルレートは `:TIMebase:SRATe?` で確認できる。

### ブロックデータの受信

バイナリ波形や画像の転送は IEEE 488.2 ブロックデータ形式で行われる。

```
#<n><length><data>
```

- `#` : ブロックデータ開始マーク
- `<n>` : `<length>` の桁数（1桁の数字）
- `<length>` : データ本体のバイト数（`<n>` 桁の数字）
- `<data>` : バイナリデータ本体

tmctlLibでは `ReceiveBlockHeader()` でヘッダを解析し、`ReceiveBlockData()` でデータ本体を受信する。  
大容量データの場合はループで分割受信すること。

### エラー確認

コマンドが正しく実行されたか確認するには：

```python
error = query(':STATUS:ERROR?')
# 正常時: '0,"No error"'
# エラー時: '<コード>,"<メッセージ>"'
```

コマンド送信後にエラーが発生した場合、後続のコマンドが無視されることがある。  
デバッグ時はこまめにエラーを確認すること。

### その他

- 波形データのASCII転送は遅い（数十万点で数十秒かかる）。大量データはWORDフォーマットを推奨
- `:COMMunicate:LOCKout ON` でパネル操作を禁止できるが、ローカルボタンは常に有効
- ファイル名に日本語を使う場合、文字化けする可能性がある。ASCII文字のみ推奨
- `*RST` を実行すると全チャンネルの設定が初期化される。必要な場合は設定保存してから実行すること
