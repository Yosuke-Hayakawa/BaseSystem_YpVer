# projects/ — 製品別プロジェクト

製品ごとの設定・リファレンス・出力先を格納するフォルダ。

## 919Dについて

`919D/` は **新プロジェクト作成時のテンプレート（見本）** として残してある。

- 新しい製品の解析を始めるときは `919D/` をコピーして中身を書き換える
- 919D自体が不要なら削除してよい（スクリプトや他ファイルへの影響は一切ない）

## 新プロジェクトの作り方

```powershell
# 1. 919Dフォルダをコピーしてリネーム（エクスプローラーでコピペでもOK）
Copy-Item -Recurse 919D <新製品名>

# 2. config.yaml を開いて設定を書き換える
```

## フォルダ構成

```
projects/<製品名>/
├── config.yaml    # 製品固有設定（必須）
├── docs/          # 製品固有リファレンス（NG番号一覧、仕様メモ等）
├── output/        # NG解析結果レポートの出力先
├── data/          # ログデータ配置先（.gitignore対象）
└── 要解析/        # 特異パターンデータ（.gitignore対象）
```

## config.yaml の主な設定項目

| 項目 | 内容 |
|------|------|
| `project_name` | プロジェクト表示名 |
| `xfileconv` | XFileConv.exeのパス |
| `temp_dir` | 一時ファイル保存先 |
| `test_set` | テスト識別パターン（ステップ列名、NG列名等） |
| `ng_names` | NG番号→説明の対応表 |

詳細は `919D/config.yaml` 内のコメントを参照。

## 別PCへの転用

1. `projects/<製品名>/` フォルダごとコピー
2. `config.yaml` 内のパス（`xfileconv` 等）を転用先の環境に合わせて書き換え

それだけで動く。
