## 2. ステロクロック制御の信号フロー

### 2.1 正常フロー（アンロック動作）

```
照合ECU                   ID-BOX                    ステロクECU
   │                        │                          │
   │─IDT1S01(STUL=1)──────→│                          │
   │  ステアリングアンロック要求  │                          │
   │                        │─(内部処理)─→              │
   │                        │  IMOSTUL=1               │
   │                        │  (イモビ照合OK時)          │
   │                        │                          │
   │                        │─IDT1S11/IDB1S01────────→│
   │                        │  アンロック命令             │
   │                        │                          │
   │                        │                          │─(モータ駆動)
   │                        │                          │
   │←─STL1S01(SIUL=1)──────────────────────────────────│
   │  アンロック確定                                     │
```

### 2.2 NG30発生パターン（推定）

```
照合ECU                   ID-BOX（断線？）            ステロクECU
   │                        ╳                         │
   │─IDT1S01(STUL=1)──→ ╳（届かない）                │
   │                        ╳                         │
   │←STL1S01(SILK=1)───────────────────────────────────│
   │  ロック継続 → イモビセット維持 → IND点滅 → NG30  │
```

### 2.3 ステロク情報①::ロック/アンロック確定（STL1S01 PID=0x33）

| 信号名 | 意味 | uf.c変数 |
|--------|------|----------|
| ロック確定 (SILK) | 1=ロック状態 | in.Lin_SILK |
| アンロック確定 (SIUL) | 1=アンロック状態 | in.Lin_SIUL |

**NG81判定コード（コメントアウト中）:**
```c

{
    ng_set(81);  // ステロクロック異常
```

## 4. ID-BOX LIN通信仕様（04_IMB-LIN-TR-020-B_ja.docより）

### 4.1 ID-BOXが送信するフレーム

| フレーム | 主要信号 | 意味 |
|----------|---------|------|
| IDB1S01 (PID=0x31) | IDB_SLP | スリープ可能状態 |
| | IDB_WKUP | ウェイクアップ送信状態 |
| | IDB_DATA | データ確定状態 |
| | ENG | EFI通信(コード①受信) |
| | ステロクロック要求受信 | 照合ECUからのSTLK受信確認 |
| | ステロクアンロック要求受信 | 照合ECUからのSTUL受信確認 |
| IDB1S02 (PID=0x39) | Lコード登録状態 | |
| | ID-BOX-照合(ステロク)データ | 暗号系 |

### 4.2 ID-BOXが受信する信号

| 信号 | 記号 | 送信元 | 意味 |
|------|------|--------|------|
| エンジン始動状態 | EGST | 照合ECU→ID-BOX | エンジン始動許可/禁止 |
| 12ビットコード送信要求 | BTCST | 照合ECU→ID-BOX | |
| ステアリングロックアンロック要求 | STUL | 照合ECU→ID-BOX | 多機関照合OK後のアンロック要求 |
| ステアリングロックロック要求 | STLK | 照合ECU→ID-BOX | 多機関照合OK後のロック要求 |

### 4.3 断線検知の仕組み

仕様書上、ID-BOX自体に**LIN断線検知の明示的な記述はない**。ただし:

1. **LINバススリープ状態の管理**: IDB_SLP/IDB_WKUPの状態遷移で通信状態を管理
2. **受信途絶検出**: LIN通信メイン仕様（060-A）に「LIN受信フレーム途絶検出」機能あり
   - 受信途絶検出時間経過 → フェール値を設定
3. **Physicalバスエラー検出**: バスエラー発生時 → LINミドルウェアを強制スリープ
4. **照合ECU側(uf.c)の検出**:
   - NG90: LIN通信停止（Sleep中にStatusCounterが変化 → 異常）
   - NG91: LIN通信途絶（Wake中に250ms以上StatusCounter変化なし → 異常）

## 5. 断線チェックポイント候補（優先度順）

### Priority 1: 最優先チェック

| No | チェックポイント | 理由 | 確認方法 |
|----|----------------|------|----------|
| **P1** | **ID-BOXのLIN_BUS端子の物理接続** | ID-BOX断線の直接確認 | テスター/オシロでLIN波形確認 |
| **P2** | **ID-BOXコネクタの嵌合状態** | コネクタ接触不良が最もよくある断線原因 | 目視＋抜き差し確認 |
| **P3** | **LINバスのID-BOXフレーム(IDB1S01/02/03)受信有無** | ID-BOXが応答しているか | CRAMASのLINモニタ or ioset.ios経由でUF変数確認 |

### Priority 2: ステロクECU側

| No | チェックポイント | 理由 | 確認方法 |
|----|----------------|------|----------|
| **P4** | **ステロクECU Pin36 LIN_BUS端子の波形** | ステロクECUがちゃんとLIN通信できているか | オシロでLIN波形確認 |
| **P5** | **ステロク情報①のSILK/SIUL値** | ロック確定が継続していないか | in.Lin_SILK / in.Lin_SIUL をモニタ |
| **P6** | **ステロクECU Pin25(+B)の電圧** | 電源断でモータ不動作の可能性 | テスターで電圧確認 |

### Priority 3: 共通バス

| No | チェックポイント | 理由 | 確認方法 |
|----|----------------|------|----------|
| **P7** | **LINバスの終端抵抗** | 断線時にバスレベルが不安定に | テスターでバス抵抗測定 |
| **P8** | **LINバスのレセッシブ電圧** | 正常時≒+B付近、断線時=異常 | テスターでDC電圧確認 |
| **P9** | **Lin_StatusCounterの変化** | LIN通信全体の生死判定 | uf.cのval.Lin_SC_timer確認 |

### Priority 4: 照合ECU側

| No | チェックポイント | 理由 | 確認方法 |
|----|----------------|------|----------|
| **P10** | **照合ECUのLIN端子の波形** | マスターからヘッダが正常に出ているか | オシロでLIN波形確認 |
| **P11** | **照合ECU→ID-BOXのSTUL/STLK送信確認** | アンロック命令が出ているか | IDT1S01フレームモニタ |
| **P12** | **照合ECU代行モード(IDT1S11)の動作確認** | ID-BOX無し時に代行で動作するか | IDT1S11フレーム確認 |

## 8. 判定サマリ

### NG30発生の推定メカニズム

```
ID-BOX断線
  → ID-BOXがSTUL(アンロック要求)を受信できない
  → ステロクECUにアンロック命令(IMOSTUL)が中継されない
  → ステロクECUはロック状態を維持（SILK=1, SIUL=0）
  → 照合ECUはイモビセット状態を維持
  → インジケーター点滅継続
  → NG30（インジケーター異常）
```

### 信頼度: **MEDIUM**

理由:
- LIN仕様上のフローは整合するが、物理的なピン・配線の確認が回路図画像を読めないため未完了
- 照合ECUの「代行ID-BOXモード」で断線時もステロクに命令が届く可能性を排除できていない
- 実機での波形確認が必要
