## 3. NGカスケード構造

**結論: 完全なドミノ倒し型。9個のNGは1つの根本原因から連鎖。**

```
[根本原因] ECUがドアアンロックCAN信号を送信しない
    │
    ├─→ [NG50] case22: 5秒タイムアウト（ドアアンロック未検出）
    │
    ├─→ [NG70/NG72] case23: CLG1/CLG2の瞬間的電圧降下
    │   └ CLG2: 11V → 0.2V → 11V (パルス状の不安定動作)
    │   └ CLG1: 11V → 5.5V → 0.3V → 11V (493ms後に同様の挙動)
    │   └ ※ドアロック状態でのアンテナ駆動パターンが通常と異なる可能性
    │
    ├─→ [NG20] case30: エンジン始動操作中にCan_BZ=1（ブザー吹鳴）
    │   └ ECUがキー未検出状態でSSW操作 → 警告ブザー
    │
    ├─→ [NG12] case30: 30秒タイムアウト（エンジン始動失敗）
    │   └ AccdIn=0.06〜0.29V, IgrdIn=0.06〜0.32V（ACC/IG未ON）
    │   └ ECUがキーを認証できず電源投入を拒否
    │
    ├─→ [NG80] case31: メーター表示異常（SWR1=6≠8, SWR3=0≠7）
    │   └ エンジン未始動状態の値が「始動後の期待値」と不一致
    │
    ├─→ [NG92] case31: PFW=0≠19
    │   └ エンジン未始動のためLIN PFW出力が0（始動後は19が期待値）
    │
    ├─→ [NG30] case31: インジケータ点灯（IndIn=9.1V）
    │   └ エンジン始動判定中にIND点灯 → 始動中は消灯が正常
    │   └ ECUがキー未認証状態を表示している可能性
    │
    └─→ [NG75] case34: CLG5=6.9V（室内アンテナ異常活性化）
        └ 乗車準備中にCLG5が6V超 → 通常0V固定が正常
        └ ECUがキー探索動作を続行している可能性
```

## 4. 原因推定

### 最有力仮説: V10_Volt（電源電圧）= 6.0V 固定

| 信号 | 実測値 | 正常値 | 状態 |
|------|--------|--------|------|
| V10_Volt | 6.0V | 12V | **全行6.0V固定** |

**V10_Volt=6.0Vが全試験を通じて一定。** これは以下の可能性を示す:

1. **テスト設定の問題**: CRAMAS側の電源電圧設定が6Vになっている
2. **電源ボード異常**: AIボードの+B入力が正しく供給されていない
3. **低電圧試験シナリオ**: 意図的な6V設定だが、ECUの動作限界を超えている

6V電源でECUのCPUは動作するが（Main/Sub Dutyは正常範囲）、以下の機能が低電圧で動作しない可能性:
- CANトランシーバの送信機能
- スマートキーRFモジュール
- ドアロックアクチュエータ制御

### 対向仮説

| 仮説 | 可能性 | 根拠 |
|------|--------|------|
| CAN信号マッピングエラー | MEDIUM | LSWD等が全行0はマッピングミスの可能性あり |
| エミュレータ不応答 | MEDIUM | キー検出→アンロックの連鎖が一切動作していない |
| ECU初期化失敗 | LOW | CPU Dutyは正常範囲で動作している |

### 報告書2(案件2)との比較

- 案件2: 特定caseでの単独NG
- **案件4: case 22から全面的に機能不全 → 根本的に異なるパターン**
- 案件4は「ECU全体が正常動作できていない」レベルの問題
